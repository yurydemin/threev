# Бэклог доработок

> Собрано по состоянию на подготовку релиза **v0.2.0** (2026-07-14). Список накапливался по ходу разработки Этапов 1–5 (раздел «Известные риски / технический долг» плана каждого этапа) плюс пункты, всплывшие при ручном тестировании перед v0.2.0. Ничего из этого не блокирует v0.2.0 — реализация в одной из следующих версий, по мере необходимости.

## Функциональность

- **Превью видео** — сейчас предпросмотр поддерживает только изображения, PDF и текст (сознательное ограничение MVP). Видео открывается только через «Скачать».
- **ZIP-скачивание папки/префикса** — сейчас скачивание разворачивает структуру на диск как есть, без архивации.
- **Отдельный экран «История»** — сейчас история передач — это только вкладки «Завершённые»/«Все» внутри экрана «Передачи».
- **Импорт `.env`-credentials** — кнопка/drag-and-drop были в UX-макетах, формат импорта так и не определён, откладывался трижды подряд.
- **HTTP/SOCKS5-прокси** — раздел «Сетевые» в Настройках сейчас заглушка «Скоро».
- **Настройки retry attempts / connection timeout** — сейчас нет UI, только адаптивная логика внутри.
- **Выбор назначения Copy/Move через дерево папок** — сейчас только Select бакета + текстовое поле префикса, без навигации.
- **Rate-limiting на неверный мастер-пароль** — сейчас попытки не ограничены (защита от подбора отсутствует).
- **Автоблокировка по бездействию** — сознательно не реализована в MVP (см. согласованные ограничения Этапа 4); один unlock на сессию до перезапуска.
- **CopyObject/MoveObject для файлов >5 ГБ** — сейчас есть жёсткий лимит S3 API; нужен `UploadPartCopy` для больших объектов.
- **Полноценный системный трей** — сейчас переключатель «При закрытии окна» — это просто выбор «Выйти»/«Спросить подтверждение», без сворачивания в трей.

## Технический долг

- `transfer_queue.parts_completed`/`file_offset` — колонки в схеме фактически не используются (источник истины — `ListParts`/сравнение размера локального файла).
- Connection Manager и circuit breaker хранят состояние только в памяти процесса — сбрасывается при перезапуске (ожидаемо, не баг).
- Осиротевший multipart upload на стороне сервера возможен при краше между `CreateMultipartUpload` и записью `multipart_upload_id` в БД.
- Адаптивные тайминги (part size, timeout) — эвристики, не откалиброванные под реально измеренную скорость канала/RTT.
- Санитайзинг ключей при зеркалировании на диск — базовый (отклонение `..`), нет обработки коллизий «ключ и как файл, и как папка».
- In-memory кэш листинга объектов — без верхней границы/эвикции.
- `GetTextPreview` доверяет MIME-типу, присланному фронтендом (из `ListObjects`) — нет серверной проверки.
- Zeroing памяти для ключа шифрования — best-effort (`KeyBox.Clear()`), Go GC не даёт криптографических гарантий стирания.

## Инфраструктура и релиз

- **Подписанные инсталляторы** — сейчас macOS ad-hoc self-signed, Windows и Linux без подписи вовсе (нет сертификатов). Пользователи увидят предупреждения Gatekeeper/SmartScreen.
- **Реальный AWS S3 в автотестах** — сейчас интеграционные тесты идут только против MinIO; AWS-совместимость обоснована архитектурно, но не проверена эмпирически на реальном аккаунте.
- **Живое тестирование на настоящих Windows/Linux машинах** — сейчас критерий приёмки для этих ОС — успешная сборка в CI + `go test`, без ручной проверки UI.
- **`.deb`-пакет и `arm64`-сборки Windows/Linux** — сейчас только AppImage (amd64) для Linux и amd64-инсталлятор для Windows.
- **Профилирование RAM/CPU только на macOS arm64** — нет собственного baseline для Windows/Linux.
- **Зависимость от `bitnamilegacy/minio`** в интеграционных тестах CI — вынужденный выбор (официальный `bitnami/minio` закрыт Broadcom, `minio/minio` не работает как GitHub Actions service container без переопределения команды запуска).
- **`internal/profiling`'s pprof-эндпоинт** — не защищён аутентификацией; безопасен только пока `THREEV_PPROF_ADDR` не задан (по умолчанию выключен), задокументировать явно как dev-only флаг.
- **Ручная синхронизация версии** между `wails.json`, `frontend/package.json` и git-тегом — риск рассинхронизации при частых релизах (частично смягчено `check-version` job в `release.yml`).
- **Файл `LICENSE`** — пока отсутствует, решение о лицензии не принято.

## Локализация
